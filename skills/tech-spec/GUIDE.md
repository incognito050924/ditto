# DITTO tech-spec 사용 가이드

> **목적**: DITTO와 함께 기술 스펙을 작성하는 `tech-spec`을 쓰는 법 — 흐름, 질문 옵션, 옵션 고르는 법, 개인 기본값.
> **소비자**: DITTO 사용자. 개발 지식이 없어도 따라 쓸 수 있다.
> **갱신 조건**: 옵션·프리셋·설정이 바뀔 때 갱신한다. 옵션의 정확한 값·범위는 `ditto tech-spec start --help`로 언제든 확인할 수 있다.

---

## 1. tech-spec이 하는 일

사용자와 DITTO가 **함께 기술 스펙 문서를 작성하고**, 합의된 문서를 작업 지시로 굳히는 도구다. "무엇을 만들지"를 먼저 글로 합의하고 시작하고 싶을 때 쓴다. 가볍고 되돌리기 쉬운 요청에는 굳이 쓰지 않아도 된다.

문서가 원본이다. 마무리(finalize)하면 그 문서에서 작업 지시가 만들어진다 — 완료 조건·바꿀 범위·위험을 문서에 적어두면, DITTO가 그것을 검증 가능한 형태로 굳히고 실제 작업을 시작한다.

## 2. 흐름

1. **시작한다**
   ```bash
   ditto tech-spec start --work-item <wi> --doc .ditto/specs/<이름>.md
   ```
2. **DITTO가 좋은 질문을 골라 물어본다.** 당신은 답만 하면 된다 — 전문가가 봤을 맹점이나 놓친 결정을 짚어준다. 질문을 얼마나 깊고 넓게 할지는 옵션으로 조절한다(§3).
3. **섹션을 채우고 리뷰한다** — DITTO가 초안을 쓰고 당신이 증분마다 검토한다.
4. **마무리한다** (`ditto tech-spec finalize …`) — 문서가 작업 지시로 컴파일되고 실제 구현이 시작된다.

보통 당신이 직접 치는 명령은 `start` 한 줄이다. 질문을 만들고 고르고 묻는 과정은 DITTO가 알아서 구동한다.

## 3. 질문 옵션 (`start` 시점)

질문을 얼마나 깊고 넓게 할지 고른다. 안 주면 `standard`로 동작한다. 정확한 값·범위는 `ditto tech-spec start --help`로 확인한다.

| 옵션 | 짧게 | 무엇을 조절하나 |
|---|---|---|
| `--performance` / `-p` | 프리셋 | 아래 5단계 묶음 중 하나를 한 번에 고른다 |
| `--intensity` / `-i` | 강도 0~100 | 질문의 깊이·엄격함·개수를 한 다이얼로 함께 올린다 |
| `--generators` / `-g` | 생성기 수 | 한 번에 몇 갈래로 질문을 발굴할지(많을수록 맹점을 넓게 훑는다) |
| `--gate-mode` / `-m` | 답하는 방식 | `confirm`=질문을 당신에게 묻고 기다림 / `draft`=DITTO가 근거로 답할 수 있는 건 채우고, 되돌리기 어렵거나 가치 판단인 질문만 당신에게 남김 |
| `--generator-effort` / `-e` | 발굴 노력 | 질문을 만들 때 코드·자료를 얼마나 파고들지(`low`/`medium`/`high`) |
| `--max-questions` / `-q` | 질문 상한 | 너무 길어지지 않게 질문 총량을 제한(기본: 제한 없음) |
| `--max-rounds` / `-r` | 라운드 상한 | 질문 라운드 수를 제한(기본: 제한 없음) |
| `--threshold` / `-t`, `--granularity` / `-d` | 고급 | 선정 기준·질문 세분화를 직접 조절(보통은 안 건드려도 강도가 알아서 정한다) |

**프리셋**(`--performance`) — 한 번에 고르는 5단계:

| 프리셋 | 성격 | 생성기 | 질문 강도 |
|---|---|---|---|
| `glance` | 훑어보기 — 핵심만 빠르게 | 1 | 가장 낮음 |
| `quick` | 가볍게 | 2 | 낮음 |
| `standard` (기본) | 표준 — 대부분의 작업 | 2 | 보통 |
| `deep` | 깊게 — 맹점을 넓게 | 3 | 높음 |
| `exhaustive` | 가장 철저하게 | 4 | 가장 높음 |

## 4. 어떤 걸 고를까

- **빠른 합의 / 작은 변경**: `glance` 또는 `quick`. 핵심 질문만 빠르게.
- **보통 작업**: `standard`(그냥 옵션 없이 `start`). 대부분 여기서 충분하다.
- **모호하거나 위험한 작업**: `deep` 또는 `exhaustive`. 여러 갈래로 맹점을 넓게 훑고 까다롭게 골라낸다 — 놓치면 비싼 결정일 때.
- **개입을 줄이고 싶을 때**: `-m draft`. DITTO가 답할 수 있는 질문은 근거를 달아 스스로 채우고, 되돌리기 어렵거나 가치 판단인 질문만 당신에게 묻는다.
- **인터뷰가 길어지는 게 싫을 때**: `-q`(질문 수) 또는 `-r`(라운드 수)로 상한을 둔다. 안 둬도 더 물을 게 없으면 알아서 끝난다.

## 5. 매번 안 치고 싶다면 — 개인 기본값

`.ditto/local/config.json`에 기본값을 적어두면 매번 플래그를 주지 않아도 된다:

```json
{
  "tech_spec": {
    "question": {
      "performance": "deep",
      "max_rounds": 5
    }
  }
}
```

- **명령에 플래그를 주면 그게 우선**, 없으면 이 설정값, 그것도 없으면 기본값 순으로 적용된다.
- 당신 개인 설정이다 — 팀과 공유되지 않고(이 파일은 버전 관리에서 제외된다), 없거나 잘못 적어도 기본값으로 안전하게 동작한다.
- 한 번 적어두면 보통 `ditto tech-spec start`만 쳐도 이 기본값이 적용된 채로 동작한다.
