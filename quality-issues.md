## DITTO 전반
- 메인 에이전트/planner 등과 같은 관제탑 역할의 에이전트들의 컨택스트 윈도우 관리가 너무 안 됨 (역할의 철저한 분리와 위임/보고 체계가 잘 설계 및 동작하는지 의심) > Context Rot 의심

## 의도 파악 및 방향성 결정 표면
- 의도를 임의로 줄임
  - 사용자 인터뷰에 질문 확산을 막기 위한 자가 대답 때문인지 몰라도 사용자 의도를 조용히 의도를 줄이거나 왜곡/곡해하여도 알 수 없음
  - 사용자는 결국 선택지 중에 하나를 고를 경우가 대부분이라 결국 ditto 의 방향성을 따르게 됨 (ditto 가 올바르게 이끌 수도 있겠지만 확신 낮음)
  - 사용자의 의도를 파악하여 올바르게 모호성/미흡/실수/오판/정합성/등 을 보정해 나가는 것을 기대했지만 오히려 질문을 거듭할 수록 ditto 의 편향에 빠지게 됨 (의사결정 트리처럼 전체 지도를 계속 확장/업데이트/동기화 하면서 모든 DAG 를 처리해야 끝난다면 모를까, 사용자 답변에 따라 실시간으로 노드가 추가되거나 삭제(해결)될 수 있음)
  - 사용자는 ditto 의 인터뷰 질문이 ditto 가 사용자의 어떠한 의도/모호성을 밝히려 하는지 직관적으로 알기 어려움. 답변은 하지만 어떤 맥락인지 어떤게 해소되는지 알 수 없음. 그래서 잘못된 답이나 잘 모르겠어서 아무 답변이나 고름 > 품질 저하로 직결
- 의도를 잘못 파악하여, 나중에 검증 회귀에 빠지거나 프로젝트의 품질이 낮아서 재작업 하는 경우, 갈아엎는 경우 다수 발생 (이 문제는 구현/검증 단계까지 고려 필요)
- work item 으로 종속시키고 여기에 구현까지 묶는 설계 자체를 재설계하는 것도 고려 필요 

## 표준 구현 절차
- 의도를 임의로 줄임 (자기 완결성 관점에서 최악의 사용자 경험/품질)
  - 설계 단의 문제인지 AC 추출 및 테스트 케이스 작성의 문제인지 완수 구현 결과의 품질은 둘째치더라도 범위의 확산을 제어해내지 못함. (deep-interview와 autopilot 에서 계획 단계에서 pre-mortem 까지 하는데도 후속이나, 잔여 이슈 없이 깔끔하게 한번에 끝난 적이 한번도 없음) 
  - 멋대로 슬라이스로 나누어 진행. 슬라이스로 나누는 것이 문제가 아니라 일부 슬라이스만 구현하고 사용자에게 떠넘기는 것이 문제
  - 남은 이슈나 후속을 남긴 채로 종료. 명확한 방안이 있음에도 사용자에게 책임 떠넘김
- 위의 최초 구현 범위가 사용자 의도대로 한번에 완결이 되지 않아 발생하는 이슈
  - 발생한 후속이나 남은 이슈들을 한번에 처리하려고 해도 또 다시 자기 완결성 부족으로 잔여가 남음. 이것이 반복되어 발견된 잔여가 완료되지 않고 묻히는 경우 다수, 사용자의 인지 비용이 폭증
- 후속/잔여/위험/이슈 처리를 위해 work item 과의 관계나 설계를 다시 고려하는 것도 적극 검토할 것
- 품질 위주의 autopilot 과 경량 경로도 있는걸로 알고 있는데 경량 경로는 내가 지시하여 만들었지만 잘 모름. 어떤 구조인지, 무엇이 다른지, 어떤 경우에 사용되는지 등
- 전체적으로 너무 오래 걸림 (호스트의 LLM 모델, 병렬 작업, 비용 최적화, task 및 단계 최적화, 에이전트 호출 최적화 등 전반적인 최적화 필요)

## 코드 품질 관리 절차
- ACG 로 어느 정도 코드 품질을 보장하려고 하였지만 효능감 없음. 기존의 코드가 더 나아지고 있는지, 나아지게 할 수 있는지 검증 필요
- 향후 코드베이스 온톨로지를 구축하려면, 사전에 좋은 코드베이스가 되어야 함. 위에서 ACG 가 이것을 고려하고 있는지 확신 없음 (Meta 의 pre-compute 기술 조사 필요. 다음 페이지도 조사 필요 https://engineering.fb.com/2024/12/19/developer-tools/glean-open-source-code-indexing/ · https://github.com/facebookincubator/Glean · https://glean.software/docs/angle/guide/)
- 한번이 아니고 지속적으로 코드베이스를 관리해야함 (gardening). TIDY 원칙에 따라 구조 변경이나 리팩토링은 기능 구현과 분리 필요.
