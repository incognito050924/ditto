## 1. 기능

- **이름**: 불필요한 문서 정리 (reports/ 누적 문서의 신뢰도 회복)
- **층위**: 저장소 문서 거버넌스(reports/ git-추적 문서). 코드·런타임 산출물 아님.

## 2. 요약

reports/ 아래 누적된 git-추적 문서(handoffs·design·harnesses)에서 **사실적으로 죽은 문서(명시 SUPERSEDED·명백한 중복/대체)**를 가역적으로 정리해, 남은 문서가 현재 코드·결정과 어긋나지 않고 참조 무결성이 유지되는 상태로 만든다. 목표는 파일 수 감소가 아니라 **남은 문서의 신뢰도 회복**이다.

## 3. 배경 [장]

문서가 프로젝트 내내 reports/에 쌓였고 아무도 쳐내지 않았다 — `reports/handoffs/`(3개, 1개 SUPERSEDED), `reports/design/`(~12개), `reports/harnesses/`(~15+개, 외부 1차 조사 노트 포함). 거버넌스는 이미 있다: Agent Charter §9(CLAUDE.md:277)는 모든 문서가 목적/소비자/갱신주기/삭제조건을 댈 수 있어야 한다고 정한다. 문제는 정책 부재가 아니라 적체 미해소다. 동시에 위험도 분명하다 — 문서 삭제는 빌드·테스트가 잡지 못하는 참조 무결성 회귀(끊긴 supersedes chain, 죽은 projection 링크)를 만들 수 있고, harnesses의 외부 조사 노트는 재현 불가한 1차 자료라 '오래됨=불필요'가 가장 안 맞는 부류다.

## 4. 목표

- 명시적으로 죽은 문서(SUPERSEDED 마커 보유, 또는 명백한 중복/대체)를 식별해 가역적으로 정리한다.
- 정리는 **가역 우선**: 1차 처분은 SUPERSEDED 마킹 또는 통합이며, `git rm`(워킹트리 제거)은 명백한 중복에 한정한다.
- 정리 후 reports/ 내부 상호 참조·supersedes chain·CLAUDE.md projection 링크가 끊기지 않은 상태를 유지한다.
- 남기기로 한 문서가 §9 메타데이터(목적/소비자/갱신주기/삭제조건)를 댈 수 있는지 점검한다.

## 5. 비목표 (변경 경계) [장]

- **마커 없는 stale 문서의 일괄 정리를 하지 않는다** — 나이/추론 기반 판정은 오삭제 위험이 커 이번 범위에서 제외(보수적). 명시 신호만 신뢰한다.
- **harnesses/ 외부 1차 조사 노트를 삭제하지 않는다** — 재현 불가 자료라 보존이 기본값.
- **이미 SUPERSEDED로 표시된 핸드오프를 삭제하지 않는다** — 의도된 결정 이력(감사 흔적)으로 보존한다. 마킹은 종착 상태이지 삭제 예약이 아니다.
- **CLAUDE.md projection·README·glossary 등 자동 투영 산출물을 손으로 지우지 않는다** — SoT 자동 재생이라 손삭제는 무의미하거나 drift를 만든다.
- **`.ditto/local/` gitignore 산출물·위치가 틀린(ADR-0005 위반) 문서 재배치는 이번 범위 밖**이다 — 별도 판단.
- 재사용 정리 규칙·자동화를 만들지 않는다(Charter §4-3: 한 줄 규칙을 프레임워크로 키우지 않음).

## 8. 계획 (Plan) [단]

> ⚠ 비구속 설계 힌트. 더 나은 절차로 대체 가능.

1. 대상 후보 수집: `reports/`에서 SUPERSEDED 마커 grep + 명백한 중복(동일 주제 다중본) 식별.
2. 각 후보의 inbound 역참조 grep(문서·코드·ADR·CLAUDE.md projection·memory) → 참조 있으면 처분 보류 또는 링크 갱신 동반.
3. 처분: 가역 우선(SUPERSEDED 마킹/통합) → 명백 중복만 `git rm`. 구조적(이동/마킹)과 삭제를 커밋 분리.
4. 검증: 죽은 내부 링크 0 확인(grep), §5 경계 밖 미변경 확인(diff), 보존 대상(SUPERSEDED·1차자료) 미손실 확인.
