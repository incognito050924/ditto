## 컨텍스트

#29(wi_260714aaq)가 done으로 닫히며 `intent.follow_up_candidates`에 3건이 후보로 남았다. 방침(사용자 지시, ADR-0009 재확인): 후속·잔여를 "보류"로 계속 띄워두지 않는다 — 각 건을 **지금 구현** 또는 **명시적 비-구현(철회조건 포함)** 또는 **올바른 주인에게 재배치**로 종결한다. 세 후보:

1. #30 완료 후 배너 노출도 advisory→hard 승격.
2. 방향 (b): glossary에 없는 미합의 용어를 산문에서 감지해 합의로 유도.
3. self-check(유출 검출)을 모든 CLI 사용자 출력으로 전면 확장.

판단 기준(ADR-0009 계승): 작아도 유의미한 효과가 있으면 구현하되, 공연히 복잡성을 높이거나 위험을 내포하면 과감히 명시적으로 구현하지 않는다.

## 결정

- **(2) 방향 (b) 미합의-용어 검출·유도 = 구현 안 함.** #29는 deep-interview에서 사용자가 "(a)만"으로 확정한 방향이고, (b)는 그 결정 및 코드에 기록된 결정을 뒤집는다.
- **(3) self-check의 전(全) CLI 출력 확장 = 구현 안 함.** 고가치 표면은 이미 덮였고, 나머지는 저가치 롱테일이다.
- **(1) 배너 노출 advisory→hard 승격 = #30 소관으로 재배치**(#29의 미완/빚이 아님). #29 D2가 "질문=차단, 배너=#30까지 advisory"로 명시 이연했고, 배너 문구가 #30에서 재작성돼 un-glossed vocab이 제거된 뒤에야 하드 전환이 안전하다. #30이 배너 문구를 재작성할 때 이 하드 승격을 함께 수행한다. GitHub #30에 노트로 이관 기록.

이로써 #29의 후속 3건은 전부 터미널 상태(비-구현 2 + 재배치 1)로 닫힌다. `intent.follow_up_candidates`는 프리즈된 기록으로 남되, 그 처분의 권위는 이 ADR이다.

## 근거 (rationale)

- **(2) 비-구현 근거 — 위험이 값을 압도하고 기존 결정과 충돌한다.**
  - *산문 스캔 = 한국어 회귀 재도입.* (b)는 "목록에 없으면 의심"이라는 열린-세계 판정이라, 결정론적 closed-list가 아니라 broad-token 매칭이 필요하다 — `question-context.ts`가 "broad `\w+`는 한국어에서 regress했다"고 명시적으로 배제한 바로 그 방식이다.
  - *기록된 결정 뒤집기.* `knowledge-bridge.ts`는 "휴리스틱 용어 추출을 하지 않고, 용어 승격 판단은 LLM KnowledgeCurator에 둔다"를 결정으로 박아뒀다(ADR-0020 성격). (b)의 자동 감지는 이를 정면으로 뒤집는다.
  - *가치는 이미 담당된다.* (b)가 노리는 이득(용어집 성장)은 knowledge 노드/`ditto knowledge-update`의 KnowledgeCurator 경로가 이미 제공한다. 새 프로세서를 만들 게 아니라 그 경로를 강화하는 것이 정합이다.
- **(3) 비-구현 근거 — #28과 동형 저가치.**
  - 대부분의 CLI 출력은 에이전트가 relay한다(LLM = 번역층). raw CLI 전 출력에 glossary-gloss를 강제해도 사람에게 닿는 값은 거의 없고 오탐 면적만 커진다. 이는 #28(CLI 정적문구 i18n)을 저가치로 드롭한 논리와 같다.
  - 사람에게 더 직접 닿는 고가치 표면(대화형 질문 면 = #29, 대화 문구 ~60곳 = #30)은 이미 덮였다. "모든 CLI 출력"은 저가치 롱테일이다.
- **(1) 재배치 근거.** 지금 배너를 하드로 켜면 #30이 아직 재작성 안 한 문구가 곧바로 게이트에 걸려 깨진다(#29 D2가 정확히 이 닭-달걀을 피하려 배너를 advisory로 둔 것). 따라서 배너 하드 전환은 #30 안에서 문구 재작성과 원자적으로 이뤄져야 하고, #29에 남길 잔여가 아니라 #30의 작업 항목이다.
- **기각된 대안 — "지금 3건 다 구현(완결=전부 실행)".** 기각. (2)를 지금 하는 것은 사용자의 D1 결정과 기록된 결정을 뒤집고 알려진 한국어 회귀를 다시 지는 것이라 완결이 아니라 해악이다. (3)은 저가치 표면 bloat다. 완결성은 "전부 실행"이 아니라 "각 건을 근거 있는 터미널 상태로 닫음"이다(ADR-0009).

## 변경 조건 (change_condition)

- **(2) 재개:** KnowledgeCurator 경로로 못 잡는 구체적 용어-드리프트가 실측되고, 그것을 결정론적(closed-list) 검출로는 해결할 수 없음이 확인될 때. 그때도 산문 broad-match가 아니라 명시된 후보 목록에 한정하는 설계를 우선 검토한다.
- **(3) 재개:** 비-LLM raw CLI 소비자(에이전트 relay를 거치지 않고 사람이 직접 읽는 파이프라인·스크립트 출력)가 실제로 등장할 때. (#28의 재개 조건과 동일.)
- **(1) 자동 소멸:** #30이 배너 문구 재작성과 함께 하드 승격을 수행하면 이 항목은 실현되어 소멸한다. #30이 won't-do로 닫히면 배너는 advisory로 영구 유지.
